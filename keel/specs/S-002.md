# Sentinel Brand-Aware Atlas Diagnostics

## Objective

Keep Sentinel thin while teaching it enough about Brandie-backed Atlas review
metadata to diagnose drift, classify failures honestly, and optionally expose
that review context where it helps audits and large review sessions.

## Why This Wave Exists

Sentinel's Atlas migration layer is now real:

- Atlas manifests and session indexes are supported
- hierarchical dashboard grouping exists
- doctor and validation can diagnose migration mistakes
- ecosystem fixtures cover Onlystack, FitKind, and Sitches-style configs

The next gap is brand-aware review context:

- Brandie can define review-pack metadata
- Atlas is gaining the ability to consume and render that metadata
- Sentinel currently cannot distinguish missing screenshots from missing review context

## In Scope

- brand-aware Atlas fixture examples
- doctor diagnostics for missing or stale Brandie review-pack references
- optional dashboard or review-output rendering of brand context
- validation messaging that separates missing screenshots from missing review metadata

## Out Of Scope

- turning Sentinel into the primary review UI
- making Sentinel own Brandie assets or Atlas manifests
- product-specific brand logic hardcoded into Sentinel

## Definition Of Done

A task in this wave is done only when:

- Sentinel can explain a real brand-aware failure mode more clearly than before
- the behavior is covered by fixtures, tests, or docs
- Brandie remains the metadata source and Atlas remains the main review product
